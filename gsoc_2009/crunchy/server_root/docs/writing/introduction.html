Writing Tutorials: Introduction

So you wrote a Python tutorial, either about the language or about some library (like pygame) or module. Congratulations, this is hard work. If your tutorial was written in html, with relevant Python code inside <pre> elements, then Crunchy can be used to interact with it. However, with just a tiny bit more work, you can make the interaction with Crunchy much richer.

VLAM

To accomplish its magic, Crunchy uses VLAM. Well ... Crunchy can even do its magic without VLAM ... but not as well, and since you wrote a tutorial, you might as well take full advantage of Crunchy's capabilities. If you think you know what VLAM stands for, and you haven't read this tutorial before, I would bet that you are wrong. This is because VLAM is an acronym that André made up. It stands for Very Little Additional Markup.

Very Little Aditional Markup, or VLAM, is essentially just an extended set of attributes for some common HTML tags. Any .htm or .html file [ and even some reStructuredText files] loaded through Crunchy will have its VLAM code processed.

The basic idea that motivated VLAM was to make it as easy as possible to transform traditional web-based tutorials into Crunchy tutorials, without changing the way that the same tutorials appeared when viewed in a normal browser without Crunchy doing its magic. Additives?

Most often, the explicit additive (VLAM) used by Crunchy will make use of the title attribute of a given html element as follows:

<pre title="some clever stuff">
... some Python code ... 
</pre>

If you are ready, you can go to the next page and learn some more about the explicit additive (vlam) used for each Crunchy interactive elements.
