How to make Crunchy Tutorials

As of version 0.8, transforming "normal" html files into Crunchy tutorials could hardly be any easier. What you need is the following:

  • A W3C compliant html file, with a filename ending with ".htm" or ".html".
  • A file in which all Python code is formatted inside <pre> tags with proper indentation (starting on the left margin). Such code can represent an interpreter output session.

Assuming you have such a file (let's call it tutorial.htm). Browse for it using the input box below and prepare to load it.

Load tutorial for editing

After loading it, you will find that crunchy has identified each <pre> tags and offer you the possibility to add some interactive elements. For each <pre>, when you make changes, click on the "Record changes" button. When you are done with all the changes for that tutorial, click on the "Update" button, at the lower right of the screen. What will happen then is the following:

  1. Crunchy will create a new file with a name where "_new" will be added before the extension; so, in this fictituous case, it would be tutorial_new.htm.
  2. Crunchy will make a copy of your original file into this new file with the addition of the special markup.
  3. Crunchy will then display that new file, ready for further editing. From that point on, any new changes you make (after clicking on the "Update" button) will be recorded in tutorial_new.htm.

That's it!

If you want some more details about the special markup, look under the menu (Tutorials -> Making Crunchy Tutorials) at either VLAM summary or VLAM examples.
