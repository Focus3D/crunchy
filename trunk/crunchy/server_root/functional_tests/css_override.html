Crunchy css test

A tutorial writer should be override the Crunchy css default, either by specifying some <style> instructions in the file or by linking to an external style sheet. This file is intended to test these features.

The following code should be styled by Crunchy, with the default overriden so as to clearly stand out (the goal is not to be esthetically pleasing!):

  • the Python keyword are in yellow on a black background; this is done by some embedded styling command in the head of this file.
  • the Python string is black on a yellow background; this is done by linking to an external style sheet.

Note Crunchy messes with Pygments' class names - it only allow one style per page, and renames the css class crunchy_pygments. This is to allow overriding the default style. Note that crunchy_pygments is the generic class name; there are specific class names for keywords (k), strings (s), prompts (gp), tracebacks (tb), etc. Just view the source of a styled page for reference, or go to Creating a new Pygments style.
