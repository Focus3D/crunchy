Colourizer: special characters test

Because of the way ElementTree parses xml strings, special care has to be given to the treatment of 3 characters: "&", "<" and ">". This is been done inside colourize.py, and this page verifies that it is handled properly. (If you see this page, chances are that it works; previous versions were just being displayed as a blank page as ElementTree could not parse the output).

Note: Unit tests need to be implemented for this.

Inside <code> element

First, using the special notation for html characters, i.e. "&amp;", "&lt;", "&gt;"

  • unprocessed element:
    if 0 < x and x > -1: print "neat & tidy"
  • Same as above, but with explictly added markup "by hand", not following the usual style, so that we can later verify that Crunchy removes the existing markup to replace it by its default:
    if 0 < x and x > -1: print "neat & tidy"
  • same input as first one, but processed by Crunchy:
    if 0 < x and x > -1: print "neat & tidy"
  • Same input as second one, but processed by Crunchy:
    if 0 < x and x > -1: print "neat & tidy"

Next, using the html characters "as is", i.e. "&", "<", ">"

  • unprocessed element:
    if 0 < x and x > -1: print "neat & tidy"
  • Same as above, but with explictly added markup:
    if 0 < x and x > -1: print "neat & tidy"
  • same input as first one, but processed by Crunchy:
    if 0 < x and x > -1: print "neat & tidy"
  • Same input as second one, but processed by Crunchy:
    if 0 < x and x > -1: print "neat & tidy"
