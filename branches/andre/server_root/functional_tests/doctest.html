Crunchy Doctest Test

Here's a Doctest box:


>>> p = Animal('Pooh')
>>> p.name
'Pooh'
>>> p.addFriend('Piglet')
>>> p.friends
['Piglet']

Try entering some code in the editor above so that it satisfies the doctest.

This should generate a syntax error if using a version of Python older than 2.5:

class Animal():
    pass

This should compile but not satisfy the doctest:

class Animal:
    pass

And this should work:

class Animal:
    def __init__(self, name):
        self.name = name
        self.friends = []
    def addFriend(self, friend):
        self.friends.append(friend)

Bugs

  • There are no known bugs specific to doctests. However, traceback messages, which are the normal Python ones, are not very friendly (unlike the old Crunchy version) and are not currently translatable.
