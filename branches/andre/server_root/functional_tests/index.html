Error handling

  • Broken html tree: The new (0.9+) version of Crunchy can now display pages with some errors in it, but not with Python 3.0; previously, when errors were found (such as missing closing tags) it was only displaying a blank page. Ideally, it Should display the original page with an error message (in a popup window?) A previous version displayed a custom alert (on top of a blank page); somehow this is now broken - but is probably not the behaviour we wanted anyway.
  • Non-existent file: currently displays a custom page whose appearance could be improved upon (perhaps with images, in a humourous way) and which should be translated to the default language - when the i18n features are working.
  • Empty file: file with no head nor body; these will be added by Crunchy, together with a very visible warning.

Others

  • Not a test as such, but the Roadmap document is something that should be looked at periodically and updated as we go along.
  • IPython is a fairly advanced replacement for the normal Python console. Unfortunately, it messes up Crunchy rather badly when used as an alternative shell. Assuming you have it installed on your system, you can try it out.
  • Sound: The old Crunchy used to have some minimal sound support on Linux and Windows. However, this was not suitable for any serious demonstration and the sound capabilities are better provided by specialised libraries.
  • As an aid to developing a new sanitize module used internally by Crunchy, a special plugin was created and can be tested. However it should be only of interest for Crunchy developers.
  • A new plugin is being developed in order to do automated code checking in documentation. To see one failing and one passing test: one failing test; you can also see what happens when all tests pass: passing tests
