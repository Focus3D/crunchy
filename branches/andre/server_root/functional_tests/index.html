Error handling

  • Broken html tree: currently displays a blank page. Should display the original page with an error message (in a popup window?) A previous version displayed a custom alert (on top of a blank page); somehow this is now broken - but is probably not the behaviour we wanted anyway.
  • Non-existent file: currently displays a custom page whose appearance could be improved upon (perhaps with images, in a humourous way) and which should be translated to the default language - when the i18n features are working.
