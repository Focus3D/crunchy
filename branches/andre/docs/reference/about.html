A long time ago...

In the beginning, there was IT. IT was nice and IT was the name suggested by André Roberge's 13 year old daughter Evelyne after he showed her and explained to her that IT was designed to produce Interactive Python Tutorials. However, IT was found to be slightly uninspiring as a name. After asking on the edu-sig list for potential names, IT became Crunchy Frog. Crunchy Frog (it probably was more of a Tadpole back then) grew slowly until Johannes Woolard joined as a developper, thanks to Google's financial support during the Summer of Code 2006. As the Frog matured and another project was found with the same name, natural selection took its course and, as of version 0.7, this project was renamed "Crunchy".

We use only the finest baby frogs, dew-picked and flown from Iraq, cleansed in the finest quality spring water, lightly killed, and then sealed in a succulent Swiss quintuple smooth treble cream milk chocolate envelope, and lovingly frosted with glucose.
From the Monty Python Crunchy Frog sketch.

Crunchy only uses the finest computer language (Python) to process html pages (cleansed by the excellent htmlTidy program) to which eye-pleasing styling options are added with a sprinkle of javascript for added interactivity; all code is carefully crafted from both sides of the Atlantic, with the goal of transforming your reading of Python related tutorials into splendidly entertaining interactive sessions within a Gecko based browser, thanks to the light addition of VLAM.

As of version 0.8, Crunchy uses significantly more javascript through the inclusion of Christophe Dolivet's EditArea, as an embedded Python editor.

VLAM

To accomplish its magic, Crunchy uses VLAM. If you think you know what VLAM stands for, and you haven't read this tutorial before, I would bet that you are wrong. This is because VLAM is an acronym that André made up. It stands for Very Little Additional Markup.

The basic idea that motivated VLAM was to make it as easy as possible to transform traditional web-based tutorials into Crunchy tutorials, without changing the way that the same tutorials appeared when viewed in a normal browser without Crunchy doing its magic. (You may have to read the previous sentence more than once as it is admittedly too long.)

As of version 0.6, all VLAMed files are only using W3C compliant markup elements.
