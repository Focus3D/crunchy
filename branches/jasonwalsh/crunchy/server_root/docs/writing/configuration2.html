Enabling preferences settings

In the category of you're never going to need this [just kidding], is the ability to create a page where the user is allowed to set her preferences using graphical elements. This is what we use in Crunchy's standard user's preference page. Here's an example of this where you can change the value of "doc_help".

access to doc_help

This was done with the following html code

<div title="preferences doc_help">irrelevant text</div>

More than one value?

Multiple values are also allowed; all you need is to separate them by spaces. For example, this code:

<div title="preferences language doc_help dir_help">some text</div>

will yield the following:

access to a few options

Note that they have been re-arranged to be in alphabetical order.

Grouping by type

As you know, there are three types of user preferences:

  • Those with boolean values True or False 
    <div title="preferences boolean">...</div>
    ...
  • Multiple choices 
    <div title="preferences multiple_choice">...</div>
    ...
  • User defined options 
    <div title="preferences user_defined">...</div>
    ...

As you could see, Crunchy allows you to only display one type at a time otherwise we would not have raised the subject
