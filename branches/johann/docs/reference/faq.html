What is Crunchy Frog?

Crunchy Frog is an HTTP Proxy Server, i.e. an application that requests html pages, processes them, and displays them inside a browser window. In more practical terms, Crunchy Frog transforms a normal looking html page into an interactive Python session. Currently, it can insert any number of the following interactive objects within a single html page:

  • A Python interpreter.
  • A very primitive editor in which you can write some code and have it executed by Python.
  • A special editor mode that automatically runs some pre-defined doctests on the code.
  • A simple (non-interactive) graphics interface.
  • A simple cross-platform sound API, accessible from any code executed in Crunchy.

What is VLAM?

VLAM stands for Very Little Added Markup. It is a W3C compliant way of encoding information in html files. There are currently three different types of markup we use:

  • <meta name="vlam" content="...">
  • <pre title="some vlam commands">
  • <span class="vlam_comment" style="display: none">
More information can be found here.


Why doesn't VLAM define its own proprietory xml attributes?

Because we like to code webpages that show up as correct in HTML Tidy. We believe that VLAM should be (and is) as non-disruptive as possible to your web-coding experience.

Ideally we would like to expand the xhtml DTD to include some vlam attributes like this:

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/xhtml1-strict.dtd" [ <!ATTLIST div vlam CDATA "none"> ]>

But sadly this breaks Firefox (yes, we've filed a bug...).

What exactly is/is not supported on each platform?

Most of Crunchy should work on anything that runs Python and has a modern browser! However, there are some features that are specific to a smaller set of platforms.

Sound is only supported on the following platforms:

  • *nix: using GStreamer.
  • *nix using OSS.
  • Windows: Using the built-in winsound API.

In addition, it should be noted that by modern browser we mean Firefox. At present Crunchy Frog is only developed and tested on Firefox. We have tried it with both Internet Explorer and Opera, but it does not work. If anyone is willing to help in any way, let it be by testing or, perhaps, by helping us figuring out how to make it work with IE or Opera (meaning: working with javascript), or testing it with some other browsers, or writing Crunchy Frog compatible Python tutorials, please get in touch.

Can I start Crunchy with my own starting tutorial page?

You can start Crunchy with any page as the starting point by creating a Python script containing the following two lines:

import crunchy
crunchy.run("full_path/tutorial.html")
