Language negotiation

Abstract

Some websites implement a language negotiation system, to provide a content in the visitor's language. This is a part of the content negotiation, and use the Accept-Language header.

Crunchy can forward received Accept-Language headers to a remote site, to get it in the same language than you would just using the browser instead of Crunchy. The default state for Crunchy is:

crunchy.forward_accept_language = True # enabled by default!

which you can disable:

crunchy.forward_accept_language = False

How to choose the prefered language within a browser?

Web browsers allow to set a list of prefered languages, sorted by priority. For Mozilla Firefox, the choice is available in Edit > Preferences > Content > Languages (or Tools > Options > Content >Languages, etc., depending of the system).

Sites known to use language negotiation

  • Mozilla Europe
  • The Apache documentation
  • MoinMoin wikis localize some part of the interface (like the login button)
